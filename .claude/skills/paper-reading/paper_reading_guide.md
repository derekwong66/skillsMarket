# 文献阅读处理指南

当用户提供文献后，按照以下步骤进行分析：

## 步骤 0：验证内容获取 ⚠️

**在开始分析前，务必确认文献内容已正确获取！**

### 内容获取验证检查清单

1. **检查 Read 工具返回值**
   - 如果返回只有 `{"type": "document", "source": {...}}` 等元数据，说明文本内容未正确提取
   - 正确的 PDF 读取应包含实际的文本内容

2. **PDF 读取失败的备用方案**
   - 如果是 arXiv 论文，使用 `mcp__web_reader__webReader` 从 arXiv URL 获取
   - 使用 `Bash` + `strings` 命令检查 PDF 元数据中的标题信息
   - 提示用户提供论文的文本内容或其他格式的文件

3. **交叉验证论文信息**
   - 检查论文标题与用户预期是否一致
   - 如果不匹配，立即停止并告知用户
   - 不要基于不完整或错误的信息进行分析

### 常见问题排查

| 症状 | 可能原因 | 解决方案 |
| :--- | :--- | :--- |
| Read 返回元数据但无文本 | PDF 解析失败 | 使用 webReader 获取 |
| 论文标题与预期不符 | 文件路径错误 | 让用户确认文件路径 |
| 内容为乱码或空 | PDF 格式问题 | 请求用户提供其他格式 |

## 步骤 1：获取文献内容

根据用户提供的方式获取文献：
- **文本内容**：直接使用提供的文本
- **文件路径**：使用 Read 工具读取文件（支持 PDF、Markdown、TXT）
- **URL**：使用 WebFetch 或 mcp__web_reader__webReader 获取内容

## 步骤 2：提取基本信息

从文献中提取以下信息：
- 文献题目
- 作者姓名
- 文献来源（期刊、会议、年份等）
- 摘要（总结核心内容）
- 关键词
- 重要指数（根据用户研究领域判断）

## 步骤 3：分析论文结构

识别并概述论文的各个部分：
1. 引言部分 - 研究背景和问题
2. 相关理论与文献综述 - 前人研究成果
3. 研究方法部分 - 实证或理论研究方法
4. 研究结果部分 - 数据和发现
5. 讨论与分析部分 - 结果解释
6. 结论与展望部分 - 总结和未来方向

## 步骤 4：深入分析

### 研究背景
- **宏观背景**：社会、经济、科技等大环境因素
- **微观背景**：具体领域现状、问题、挑战

### 研究目的
- **总体目的**：研究的总体目标
- **具体目的**：细化的可操作目标

### 研究方法
- **方法选择依据**：为什么选择这种方法
- **方法详细描述**：
  - 实证研究：样本选择、数据收集、数据分析
  - 理论研究：理论构建、逻辑推导

### 研究问题
- **核心研究问题**：研究的核心主线
- **子研究问题**：分解的子问题

### 研究结论
- **主要结论概括**：最重要的发现
- **结论详细阐述**：逐问题解释

### 理论启示
- **对相关理论的贡献**：补充、修正或完善
- **对其他理论的借鉴意义**：跨学科参考价值

### 创新亮点
- **研究内容创新**：新发现、新视角
- **研究方法创新**：新技术、新设计
- **理论创新**：新框架、新突破

### 自我思考
- **对研究的评价**：科学性、合理性、严谨性
- **与用户研究的关联**：启发和借鉴意义

## 注意事项

### 内容验证（最高优先级）
1. ⚠️ **始终先验证内容是否正确获取**，再开始分析
2. ⚠️ **论文标题必须与用户预期一致**，不一致时立即反馈
3. ⚠️ **不要基于幻觉或不完整信息进行分析**，宁可请求用户重新提供

### 分析原则
1. 如果文献内容不完整，尽力从已有信息中提取
2. 如果某些部分不明确，使用"文献中未明确提及"标注
3. 重要指数根据研究领域的相关性判断
4. 自我思考部分需要结合用户的具体研究方向

### 错误处理
- 如果 PDF 读取失败，优先尝试从原始 URL（如 arXiv）获取
- 如果无法获取内容，明确告知用户并建议替代方案
- 如果发现分析错误，立即道歉并重新分析
