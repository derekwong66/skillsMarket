# 文献阅读处理指南

当用户提供文献后，按照以下步骤进行分析：

## 步骤 0：验证内容获取

**在开始分析前，务必确认文献内容已正确获取！**

### 内容获取验证检查清单

1. **检查 Read 工具返回值**
   - 如果返回只有 `{"type": "document", "source": {...}}` 等元数据，说明文本内容未正确提取
   - 正确的 PDF 读取应包含实际的文本内容

2. **PDF 读取失败的备用方案**
   - 如果是 arXiv 论文，使用 `mcp__web_reader__webReader` 从 arXiv URL 获取
   - 使用 `Bash` + `strings` 命令检查 PDF 元数据中的标题信息
   - 提示用户提供论文的文本内容或其他格式的文件

3. **交叉验证论文信息**
   - 检查论文标题与用户预期是否一致
   - 如果不匹配，立即停止并告知用户
   - 不要基于不完整或错误的信息进行分析

### 常见问题排查

| 症状 | 可能原因 | 解决方案 |
| :--- | :--- | :--- |
| Read 返回元数据但无文本 | PDF 解析失败 | 使用 webReader 获取 |
| 论文标题与预期不符 | 文件路径错误 | 让用户确认文件路径 |
| 内容为乱码或空 | PDF 格式问题 | 请求用户提供其他格式 |

---

## 步骤 1：获取文献内容

根据用户提供的方式获取文献：
- **文本内容**：直接使用提供的文本
- **文件路径**：使用 Read 工具读取文件（支持 PDF、Markdown、TXT）
- **URL**：使用 WebFetch 或 mcp__web_reader__webReader 获取内容

## 步骤 2：提取基本信息

从文献中提取以下信息：
- 文献题目
- 作者姓名
- 文献来源（期刊、会议、年份等）
- 摘要（总结核心内容）
- 关键词
- 重要指数（根据用户研究领域判断）

## 步骤 3：分析论文结构

识别并概述论文的各个部分：
1. 引言部分 - 研究背景和问题
2. 相关理论与文献综述 - 前人研究成果
3. 研究方法部分 - 实证或理论研究方法
4. 研究结果部分 - 数据和发现
5. 讨论与分析部分 - 结果解释
6. 结论与展望部分 - 总结和未来方向

## 步骤 4：深入分析

按照以下维度对论文进行深入分析：

### 一、基本信息
- **文献题目**：论文的完整标题
- **文献作者**：所有作者姓名
- **文献来源**：期刊名称、卷号、页码、发表年份等
- **摘要**：简要阐述摘要所传达的核心内容，包括研究的主要对象、大致方法以及初步结论
- **关键词**：从论文中提取的关键词
- **重要指数**：根据论文与用户研究领域的相关性评估

### 二、核心内容（论文结构）
分析论文的各个部分：
1. **引言部分**：主要介绍研究的背景、引出研究问题。可能会提及当前研究领域存在的一些争议或者空白点，为后文的研究奠定基础。
2. **相关理论与文献综述**：梳理该研究领域已有的相关理论，回顾之前的研究文献，指出前人研究的成果与不足，进一步明确自己研究的定位和创新点。
3. **研究方法部分**：详细说明本研究所采用的方法，例如是采用实证研究（实验法、调查法等）还是理论研究（模型构建、逻辑推理等），解释选择该方法的合理性和适用性。
4. **研究结果部分**：呈现通过研究方法得到的数据、现象或者结论。通常会使用图表、数据统计等方式直观地展示研究成果。
5. **讨论与分析部分**：对研究结果进行深入讨论，分析结果产生的原因，与已有研究成果进行对比分析，探讨结果的合理性和局限性等。
6. **结论与展望部分**：总结整个研究的主要结论，强调研究的重要发现，对未来研究方向提出展望。

### 三、研究背景
- **宏观背景**：从更广泛的领域角度出发，阐述与研究相关的社会、经济、科技等大环境因素的发展状况和趋势，这些因素如何影响到研究问题的产生和研究方向的设定。
- **微观背景**：聚焦到具体的研究领域或行业内部，描述该领域内的现状、场景竞争情况、技术水平、行业规范等方面的情况，指出目前面临的问题或者面临的挑战，从而引出本研究的必要性。

### 四、研究目的
1. **总体目的**：明确阐述本研究想要达成的总体目标。例如是为了解决某实际问题、填补某一理论空白、完善某个研究体系等。
2. **具体目的**：将总体目的细化为若干个具体的、可操作的目标。如验证某个假设、分析某种现象的影响因素、构建某种模型等。

### 五、研究方法
- **方法选择依据**：解释为什么选择这种研究方法而不是其他方法。考虑因素可能包括研究问题的性质、研究对象的特点、数据的可获取性、研究的时间和资源限制等。
- **方法详细描述**：
  - 如果是实证研究方法：说明样本的选择过程（来源、规模、代表性）、数据收集方式（问卷设计、访谈流程、实验变量控制）、数据分析方法（统计软件、分析方法）。
  - 如果是理论研究方法：阐述理论构建的逻辑起点和基本假设、理论推导过程（演绎法、归纳法等）。

### 六、研究问题
1. **核心研究问题**：明确提出本研究要解决的核心问题。这一问题应是整个研究的核心和主线，其他研究内容都围绕这个问题展开。
2. **子研究问题**：为了更好地回答核心研究问题，往往会分解出若干个子问题。这些子问题相互关联，共同构成对核心问题的全面解答。

### 七、研究结论
1. **主要结论概括**：总结研究得到的主要成果，以简洁明了的语言阐述研究发现的最重要的关系、现象或者规律。
2. **结论详细阐述**：按照研究问题的顺序，逐一详细解释针对每个研究问题得出的结论，包括数据支持、逻辑推理过程等。

### 八、理论启示
1. **对相关理论的贡献**：分析本研究对所在研究领域已有的相关理论有哪些补充、修正或者完善的地方，是否提出了新的概念、模型或者理论关系。
2. **对其他理论的借鉴意义**：探讨本研究的结论或者方法是否可以被其他相关理论借鉴，为跨学科或者跨领域的理论研究提供参考。

### 九、创新亮点
1. **研究内容创新**：指出本研究在研究内容方面与前人研究相比有哪些新的发现、新的探索方向或者新的研究视角。
2. **研究方法创新**：强调在研究方法上是否有创新之处，如是否采用了新的数据分析技术、新的实验设计或者新的样本采集方式等。
3. **理论创新**：阐述本研究是否在理论构建方面有独特之处，例如提出了新的理论框架、突破了传统理论的某些局限等。

### 十、自我思考
1. **对研究的评价**：从研究的科学性、合理性、严谨性等方面对该研究进行评价，分析研究的优点和不足之处。
2. **与自身研究的关联**：如果用户正在进行相关研究或者计划开展相关研究，思考该文献对用户研究的启发和借鉴意义，如何将该文献的研究成果应用到用户的研究中。

## 步骤 6：保存输出文件

完成分析后，将格式化的 Markdown 笔记保存到 `doc` 文件夹。

### 保存位置
```
doc/
└── [笔记文件].md
```

如果 `doc` 文件夹不存在，使用 Bash 工具创建：
```bash
mkdir -p doc
```

### 文件命名规范

格式：`{年份}_{第一作者姓氏}_{简短标题}.md`

**规则说明：**
- **年份**：论文发表年份（4位数字）
- **第一作者姓氏**：英文姓氏首字母大写，中文姓氏保留原字
- **简短标题**：提取标题核心词，用下划线连接，不超过30个字符

**示例：**
```
2024_Vaswani_Attention_Is_All_You_Need.md
2023_张三_深度学习在图像识别中的应用.md
2024_Smith_Transformer_Survey.md
```

**特殊处理：**
- 标题中的特殊字符（如 `:`、`?`、`"`）替换为下划线或删除
- 多个连续下划线合并为一个
- 文件名统一使用 `.md` 后缀

## 注意事项

### 内容验证（最高优先级）
1. 始终先验证内容是否正确获取，再开始分析
2. 论文标题必须与用户预期一致，不一致时立即反馈
3. 不要基于幻觉或不完整信息进行分析，宁可请求用户重新提供

### 分析原则
1. 如果文献内容不完整，尽力从已有信息中提取
2. 如果某些部分不明确，使用"文献中未明确提及"标注
3. 重要指数根据研究领域的相关性判断
4. 自我思考部分需要结合用户的具体研究方向

### 错误处理
- 如果 PDF 读取失败，优先尝试从原始 URL（如 arXiv）获取
- 如果无法获取内容，明确告知用户并建议替代方案
- 如果发现分析错误，立即道歉并重新分析
